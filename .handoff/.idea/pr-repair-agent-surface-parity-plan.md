## Current-state findings

Live inspection confirms the earlier parity map:

- `meta init --help` exposes `claude` and `ollama`, but no `codex` command.
- `meta_cli/src/init.rs` and `meta_cli/src/main.rs` define Claude/Ollama init paths only.
- `.codex/` exists and contains config, hooks, prompts, agents, and rules, but it is hand/project
  maintained rather than generated by `meta init codex`.
- `.claude/` has a richer command surface: resume, fleet, handoff, kb commands, checkpoint, mint.
- The Codex prompt surface is smaller: meta-status, meta-worker, meta-upgrade, kb-note, codex-rust-forge.
- Core meta skill files exist in three non-identical locations:
  - `claude-plugin/skills/meta-*/SKILL.md`
  - `.claude/skills/meta-*.md`
  - `~/.codex/plugins/cache/gitkb/meta/0.1.0/skills/meta-*/SKILL.md`
- `meta-mcp` is installed at `/home/drdave/.local/bin/meta-mcp`, but Codex sessions cannot assume
  direct `meta_*` MCP tools are surfaced; repair prompts need CLI fallback.

## Decision: canonical source

For PR repair and core meta agent guidance, use this as the canonical text source:

```text
/home/drdave/Desktop/meta/claude-plugin/skills/*/SKILL.md
```

Reasons:

- It has plugin-ready `SKILL.md` shape and frontmatter.
- It is the package source consumed by plugin/cache workflows.
- It can generate Claude skill copies and Codex skill/prompt projections.

Exception: repo-specific Codex runtime details, such as `CODEX_HOME`, `CODEX_SQLITE_HOME`, and
`rtk` routing, belong in `.codex/` templates generated from a Codex-specific target template, not in
generic plugin skill text.

## Command style rule

Use this wording in generated surfaces:

```text
In this FlexNetOS meta workspace, prefer `rtk meta ...` so commands route through the meta-owned
runtime. In portable examples, `meta ...` is the command name; if `rtk` is unavailable, use `meta ...`
after verifying it resolves to the expected binary.
```

For current PR repair lane commands, use the live surface:

```bash
rtk meta git worktree ...
```

Do not generate prompts that assume top-level `meta worktree` until that alias is present and tested.

## Parity verifier concept

Command sketch:

```bash
meta parity check agent-surfaces --json
```

Checks:

1. Canonical skill source exists under `claude-plugin/skills`.
2. `.claude/skills` projections are present and current.
3. `.codex/prompts` equivalents exist for high-value commands:
   - resume
   - fleet
   - handoff
   - kb-context/status/tasks/board
   - pr-repair
4. `meta@gitkb` or successor plugin is installed/enabled for Codex if plugin cache is the target path.
5. Project `.codex/config.toml` registers `meta-mcp`.
6. `meta-mcp` executable is available.
7. CLI fallback works:
   - `rtk meta project list --json`
   - `rtk meta git status`
   - `rtk meta git worktree list --json`
8. Current command truth is reflected in docs: `meta git worktree`, not unavailable aliases.

JSON shape:

```json
{
  "schema": "flexnetos.agent_surface_parity.v1",
  "status": "drift",
  "checks": [
    {"name": "canonical_skills", "status": "ok"},
    {"name": "codex_pr_repair_prompt", "status": "missing"},
    {"name": "meta_mcp_binary", "status": "ok", "path": "/home/drdave/.local/bin/meta-mcp"}
  ]
}
```
